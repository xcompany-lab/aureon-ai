## TL;DR [AH001_009, AH001_012, AH001_025]

O processo de vendas outbound tem duas camadas: a **call SDS** (valor + qualificação) e a **call BC** (CLOSE Framework). O elemento mais inovador é o Label — nomear as objeções do cliente antes que ele as mencione. Os 3 Buckets explicam por que outbound existe: 90% do mercado é MAYBE.

---

## CLOSE Framework [AH001_012, AH001_013, AH001_014, AH001_015, AH001_016]

Framework de fechamento aplicado pelo BC na call de conversão:

```
┌─────────────────────────────────────────────────────────────────────────┐
│  C → CLARIFY                                                            │
│      Reafirmar a situação e o gap. "Então você está em X, quer Y..."   │
├─────────────────────────────────────────────────────────────────────────┤
│  L → LABEL  ⚡ ELEMENTO MAIS PODEROSO                                   │
│      Nomear 3 objeções ANTES do cliente mencioná-las.                  │
│      "As três preocupações que pessoas na sua situação têm são..."      │
│      → Quebra o frame adversarial, cliente para de defender            │
├─────────────────────────────────────────────────────────────────────────┤
│  O → OVERVIEW                                                           │
│      Mostrar a transformação completa — o que acontece quando funciona  │
├─────────────────────────────────────────────────────────────────────────┤
│  S → SELL THE VACATION                                                  │
│      Pintar o futuro emocional. Vender a sensação, não o produto       │
├─────────────────────────────────────────────────────────────────────────┤
│  E → EXPLAIN                                                            │
│      Voltar às objeções labeladas e resolvê-las uma a uma              │
└─────────────────────────────────────────────────────────────────────────┘
```

**Por que o L funciona:** O cliente entra na call com defesas ativas. Quando você nomeia suas objeções antes dele, você sinaliza que as entende — e que já pensou nas respostas. O frame muda de "vendedor vs cliente" para "especialista ajudando prospect".

---

## Call SDS: Valor antes de Venda [AH001_009, AH001_010, AH001_011]

A call do SDS não é uma call de vendas — é uma **call de valor**. 30 minutos focados em:

1. Qualificar o prospect (STAR: Situation, Timing, Authority, Resources)
2. Entregar valor genuíno (insight, diagnóstico, framework)
3. Criar comprometimento antes da transição para BC

**Princípio:** Valor upfront antes de venda reduz resistência. O prospect chega ao BC já convicto de que o problema é real e que a empresa entende dele.

## 3 Audience Buckets [AH001_025, AH001_026, AH001_027]

```
              ┌────────────────────────────────────────┐
  YES ██ 3%   │ Prontos para comprar agora             │ → Capturados por INBOUND
  NO  ██ 7%   │ Nunca vão comprar                     │ → Ignore
              ├────────────────────────────────────────┤
  MAYBE       │                                        │
  ████████    │ 90% do mercado disponível              │ → CAPTURADOS POR OUTBOUND
  ████████    │ Convertem com oferta/timing certos     │
  ████████    │                                        │
              └────────────────────────────────────────┘
```

Empresas que só fazem inbound estão disputando 3% do mercado. Outbound captura os 90% que nunca buscariam a solução por conta própria.

---

## Consenso [AH001_009, AH001_010, AH001_025]

**Valor upfront antes de venda reduz resistência.** O prospect que recebeu algo genuinamente útil no encontro com o SDS chega ao BC muito menos defensivo. A venda começa antes do BC entrar em cena.
